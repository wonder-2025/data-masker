# Data Masker 用户测试指南

## 测试环境

**系统要求：** Windows 10/11

**运行方式：**
```bash
# 开发模式
cd /root/.openclaw/workspace/data-masker
npm run tauri dev

# 或者使用构建后的版本
npm run tauri build
# 安装包位于 src-tauri/target/release/bundle/
```

## 一、文件处理功能测试

### 1.1 Word 文档测试

#### 测试文件准备
创建测试文件 `test.docx`，包含以下内容：
```
个人信息测试文档

姓名：张三
手机号：13812345678
身份证：110101199001011234
邮箱：zhangsan@example.com
银行卡：6222021234567890123

地址：北京市朝阳区某某小区1号楼101室
IP地址：192.168.1.100
护照号：G12345678
车牌号：京A12345
```

#### 测试步骤
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 启动应用 | 显示首页，统计卡片正常 |
| 2 | 点击"选择文件" | 打开文件选择对话框 |
| 3 | 选择 `test.docx` | 文件出现在待处理列表 |
| 4 | 点击"预览" | 显示文件内容预览 |
| 5 | 点击"下一步" | 进入规则配置页面 |
| 6 | 确认规则已启用 | 个人信息、联系方式等规则已勾选 |
| 7 | 点击"下一步：预览确认" | 显示脱敏预览 |
| 8 | 检查预览内容 | 原始内容与脱敏后内容对比显示 |
| 9 | 点击"开始处理" | 开始脱敏处理 |
| 10 | 等待处理完成 | 显示处理结果统计 |
| 11 | 点击"打开输出目录" | 打开输出文件夹 |
| 12 | 检查输出文件 | 文件存在，内容已脱敏 |
| 13 | 打开脱敏后的文件 | 敏感信息已被替换为*号 |

#### 验证点
- [ ] 文件预览正常显示
- [ ] 检测到的敏感信息数量正确（应显示约8项）
- [ ] 预览对比正常显示
- [ ] 输出文件生成成功
- [ ] 脱敏效果正确（手机号、身份证等被脱敏）
- [ ] 导出报告成功生成

#### .doc 格式测试（重要）
创建 `test.doc`（旧版Word格式），重复上述步骤。

**预期结果：** 应显示错误提示，建议转换为 .docx 格式
