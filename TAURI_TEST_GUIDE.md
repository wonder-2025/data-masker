# Tauri 应用自动化测试指南（仅检测模式）

## ⚠️ 重要变更：不再自动启动应用

### 新的工作流程

```
┌─────────────────────────────────────────┐
│  步骤 1: 用户手动启动应用                │
├─────────────────────────────────────────┤
│  方式 A（推荐）: npm run tauri dev      │
│  方式 B: 双击 data-masker.exe          │
└─────────────────────────────────────────┘
                ↓
┌─────────────────────────────────────────┐
│  步骤 2: 等待应用完全启动                │
├─────────────────────────────────────────┤
│  - 应用窗口打开                          │
│  - 调试端口开启（开发模式）              │
└─────────────────────────────────────────┘
                ↓
┌─────────────────────────────────────────┐
│  步骤 3: 运行测试                        │
├─────────────────────────────────────────┤
│  npm run test:e2e:tauri                 │
│  - 检测已运行的应用                      │
│  - 连接并执行测试                        │
│  - 不会重复启动或安装                    │
└─────────────────────────────────────────┘
```

---

## 🚀 完整操作步骤

### 第一步：启动应用

#### 方式 A：开发模式（推荐）

```powershell
# 终端 1：
npm run tauri dev

# 等待看到：
# ✓ Built in XXXms
# ✓ Running on http://localhost:1420/
# ✓ 应用窗口打开

# 保持这个终端运行
```

#### 方式 B：手动启动 .exe

```powershell
# 双击打开：
windows-build\data-masker.exe

# 如果是首次运行：
# 1. 完成安装向导（选择默认路径即可）
# 2. 等待安装完成
# 3. 应用自动启动

# ⚠️ 注意：生产构建可能不支持调试端口
```

---

### 第二步：运行测试

```powershell
# 终端 2：
npm run test:e2e:tauri

# 测试脚本会：
# ✅ 检测已运行的应用
# ✅ 连接调试端口
# ✅ 执行 8 个测试用例
# ❌ 不会自动启动应用
# ❌ 不会重复安装
# ❌ 不会关闭应用
```

---

### 第三步：查看报告

```powershell
npm run test:e2e:report
```

## 🎯 关键改进

### 之前的问题

| 问题 | 原因 |
|------|------|
| 每次都要安装 | 自动启动 .exe 触发安装向导 |
| 经常失败 | 自动启动不稳定 |
| 死循环 | 启动失败后不断重试 |

### 现在的方案

| 改进 | 说明 |
|------|------|
| **只检测，不启动** | 避免重复安装 |
| **持续等待** | 最多等待 2 分钟 |
| **进程检测** | 知道应用是否正在运行 |
| **不关闭应用** | 测试完成后应用保持运行 |

## 📊 测试脚本行为

### 检测流程

```
1. 检查进程是否运行
   ↓ 是
   "✅ 检测到应用进程正在运行"

2. 检查调试端口是否开启
   ↓ 是
   "✅ 检测到调试端口"

3. 连接并执行测试
   ↓ 完成

   "ℹ️ 应用保持运行，不会自动关闭"
```

## ⚠️ 常见问题

### 问题 1：未检测到应用

**症状**：
```
❌ 超时：未检测到调试端口
```

**解决**：
1. 确保应用已启动（检查窗口是否打开）
2. 使用开发模式：`npm run tauri dev`
3. 等待应用完全启动后再运行测试

---

### 问题 2：检测到进程但没有调试端口

**症状**：
```
✅ 检测到应用进程正在运行
⚠️ 应用正在运行，但调试端口未开启
```

**原因**：这是生产构建，不支持调试端口

**解决**：使用开发模式启动应用
```powershell
npm run tauri dev
```

---

### 问题 3：连接失败

**症状**：
```
❌ 连接失败: connect ECONNREFUSED
```

**解决**：
1. 检查防火墙是否阻止端口 9222
2. 使用开发模式启动
3. 重新运行测试

---

## 💡 最佳实践

### 推荐工作流程

```powershell
# 终端 1：启动应用（保持运行）
npm run tauri dev

# 终端 2：运行测试（可以多次运行）
npm run test:e2e:tauri

# 修改代码后，重新运行测试
npm run test:e2e:tauri

# 应用保持运行，测试可以重复执行
```

### 为什么这样设计？

1. **避免重复安装**：应用只启动一次
2. **快速迭代**：测试可以反复运行
3. **稳定性高**：用户控制启动时机
4. **灵活性强**：支持开发和生产两种模式
